Features

The full picture, inside the editor

From static analysis to live cluster plan inspection — without leaving VS Code.

Core

Local Static Analysis

Detects 30+ PySpark and Databricks anti-patterns as you type. Zero cluster required. Squiggles appear instantly for collect(), crossJoin(), unsafe writes, SQL injection, schema drift, UDF overuse, and more.

  • CODE_COLLECT_001collect() OOM risk
  • CODE_ORDER_001global orderBy bottleneck
  • CODE_CROSS_001accidental cross join
  • CODE_UDF_001Python UDF — use native F.*
  • CODE_COUNT_001count() > 0 instead of isEmpty()
30+

Dry-Run Plan Analysis

Submits a neutralized copy of your script to Databricks and returns the physical Catalyst plan — join strategies, shuffle exchanges, broadcast thresholds, and cost estimates.

Job Run Analysis

Pull any past Databricks job from the sidebar. CatalystOps reads the Spark event log from DBFS and opens an interactive DAG with operator trees and issue badges.

One-Click SSH

The Clusters panel lists every cluster in your workspace. Click SSH — CatalystOps starts it, runs setup, and opens VS Code Remote SSH on the driver. No terminal needed.

Billing Dashboard

Tracks DBU and dollar spend per period directly from system.billing.usage with a 1-hour cache. After each serverless run, fetches actual DBU consumption.

Asset Bundle Support

Detects databricks.yml, lists every spark_python_task and notebook_task in the sidebar, and validates 40+ schema rules inline — alert v2 schema, volume grants, cluster config, and more.

AI Integration

Built-in MCP Server

Ships a Streamable HTTP MCP server auto-discovered by VS Code 1.99+. Runs on 127.0.0.1:49152/mcp. Claude, GitHub Copilot, and Cursor can call CatalystOps tools directly — analyze code, fetch billing data, and run dry-runs through natural language.

analyze_pyspark run_dry_run get_billing_summary get_plan_analysis list_clusters get_job_run_plan list_bundle_tasks

How it works

From install to insight
in under five minutes

01

Install

Open a Python file

Install from the VS Code Marketplace. The moment you open a .py file, local analysis kicks in — no configuration. 30+ rules light up immediately for any PySpark anti-patterns it finds.

02

Connect

Link to Databricks

Run CatalystOps: Configure Databricks Connection from the Command Palette. The setup wizard detects Azure CLI, GCP ADC, ~/.databrickscfg, OAuth, or a personal access token — and shows only your options.

03

Analyze

Run a dry-run plan

Press ⌘⇧K to submit the current file. CatalystOps neutralizes side-effects, runs the Catalyst planner on your cluster, and returns the physical plan with join strategies, cost annotations, and fixes — in the sidebar.

MCP Integration

Talk to your Spark pipeline with Claude

CatalystOps ships a built-in MCP server that starts automatically when VS Code opens, running on a static port auto-discovered by VS Code 1.99+. Claude and other AI clients can call CatalystOps tools directly through natural language.
